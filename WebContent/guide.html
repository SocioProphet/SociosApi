What is Socios?

SocIoS is an abstraction layer for aggregating data and functionality from several social networks. The SocIoS API provides a single access point to the Rest APIs exposed by the underlying social networks. It comprises an abstraction layer on top of SNS exposing operations that encapsulate the functionality of the underlying SN APIs. For each supported SN a respective adaptor has been developed. Currently, the component provides support for the following Social Networks: FlickR, DailyMotion, YouTube, MySpace, Google+ and Instagram. Support for additional social networks can be added by implementing the adaptor interface provided.

Installation

For this project we used Eclipse Java EE IDE for Web Developers. Version: Kepler Service Release 2

Glassfish

We used:

  • GlassFish Server Open Source Edition 4.1
  • jdk1.8.0_20

In order to deploy SocIos on a Glassfish server you will need:

  • The "socios.properties" file located inside the "codeFiles" folder
  • The "srcDoc" folder
  • The "ResourcesForGlassfish" folder

Steps:

  • Create and register applications for the social networks you want to use (see next section).
  • In the "socios.properties" file replace the dummy values with your own application credentials.
  • Move the socios.properties file in: "\glassfish4.1\glassfish\domains\domain1\lib\classes".
  • In eclipse create a Dynamic Web Project and name it "SociosApi" (make sure that the correct target runtime for your glassfish is selected).
  • Copy the "web.xml" file from the "ResourcesForGlassfish" folder, in the "SociosApi\WebContent\WEB-INF" folder of your eclipse project (if a "web.xml" file already exists there, then replace it).
  • Copy all the libs of the "ResourcesForGlassfish"->"libDoc" folder in "SociosApi\WebContent\WEB-INF\lib". (Eclipse will automatically detect the new libraries and add them to the build bath).
  • Copy all the subfolders of "srcDoc" folder in the "src" folder of your project in eclipse.
  • Deploy the project ("Run As -> Run on Server").

If everything went ok, you should be able to navigate to the wsdl and the wadl:

  • http://localhost:8080/SociosApi/SociosApiSoapService?wsdl
  • http://localhost:8080/SociosApi/rest/application.wadl

Calling this url:

  • http://localhost:8080/SociosApi/rest/socios/hello

should return "hello!!".

Registering Applications with Social Networks

Socios retrieves data by calling the Restful web services exposed by the social networks. However, consuming these services requires a registered application (one for each social network supported by Socios). The procedure of creating and registering an application is more or less the same for every social network and explained in detail in each network's webpage for developers.

When an application is succesfully registered, the social network provides the developer with some confidential tokens. Afterwards, one can call the Rest API of the social network on behalf of this application, by passing these credentials as part of the http-request. In this way, the social network monitors each application's activity. Obviously, any request with invalid or no credentials is ignored by the social network's servers.

Unfortunately, the social networks only entitle a limited number of calls per hour to each registered application. Therefore, if you want to use Socios, you need to create and register one application for each social network you are interested in.

After geting your credentials, you need to create a file, name it "socios.properties", and fill in the values just like in this example ("fb" stands for "facebook", "gp" for "google plus", "ig" for "instagram" and "yt" for "youtube"). You can just leave out the variables that correspond to the social networks you don't care about.

Our copy of Socios is deployed on Glassfish Server 4.1, and our socios.properties file lies on the folder: "\glassfish4.1\glassfish\domains\domain1\lib\classes". If you also happen to work with a Glassfish server, that's where you should also save your socios.properties file, and the code will read the values when the project is deployed on the server. If you want to use another server... then you are going to need to make a few changes on the code. Good luck!
